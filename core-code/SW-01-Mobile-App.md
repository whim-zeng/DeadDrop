构建用户主入口——跨平台移动应用，按照 **入口 → 发现 → 核心** 三大流程阶段承载全部用户体验。

## 2. 技术栈与约束

| 层级 | 选型 | 理由 |
|------|------|------|
| 框架 | React Native ( Expo SDK 50+ ) | 跨平台、热更新、Geolocation 生态成熟 |
| 导航 | React Navigation v6 (Bottom Tabs + Stack) | 地图页与聊天页切换频繁 |
| 状态管理 | Zustand + TanStack Query | 地理位置与纸条列表需高效缓存 |
| 地图 | react-native-maps (Google Maps / Apple Maps) | 原生性能，支持热力图叠加层 |
| 语音 | expo-av + 录音转文字（接入 SW-03）| 语音纸条录制与播放 |
| 定位 | expo-location | 后台定位权限管理 |
| UI 组件 | React Native Paper / Tamagui | 快速实现 Material Design 风格 |
| 推送 | Expo Notifications | 统一 iOS APNs + Android FCM |

### 非功能约束
- **定位精度**: 室外 GPS ±5m，室内依赖网络定位 ±20m（50m 半径需兼容）
- **离线支持**: 最近 10 条纸条本地缓存（SQLite），无网时可阅读不可发布
- **性能**: 地图加载 50 个 Marker 不卡顿；纸条列表首屏 < 1.5s
- **隐私**: 不上传精确 GPS 到日志；位置数据仅用于空间查询

## 4. 八大功能模块详解

---

### 4.1 用户注册/匿名身份系统（入口阶段）

**核心描述**: 无需实名，生成匿名代号，保护隐私

**实现要点**:
- 零门槛进入：打开 App 即可使用，**不强制手机号/邮箱**
- 匿名代号生成算法：随机形容词 + 随机名词 + 4 位数字
  - 示例：`孤独的美食家_2847`、`深夜的失眠者_9301`、`湖边的守望者_1562`
- 可选昵称：用户可自定义覆盖匿名代号（仍不关联真实身份）
- 头像系统：不显示真实头像，使用**渐变色圆形 + 首字母/代号缩写**
- 设备指纹：后台生成 `fingerprint_hash` 用于识别同一用户（不暴露给用户）
- 隐私承诺页：首次启动展示"不收集手机号、不关联社交账号、位置仅用于附近查询"

**界面流程**:
```
启动 App
  ├── 已注册？→ 直接进入 ExploreMapScreen
  └── 未注册？
       ├── WelcomeScreen（3 页轮播：①地点网络概念 ②匿名承诺 ③探索引导）
       ├── AnonymousIdentityScreen
       │      ├── 自动生成匿名代号（可刷新重新生成）
       │      ├── 输入框：自定义昵称（可选，placeholder="给自己起个名字"）
       │      └── 头像颜色选择器（8 种预设渐变色）
       └── LocationPermissionScreen
              ├── 说明："我们需要定位来找到你附近的纸条"
              ├── 申请定位权限
              └── 可选"稍后设置"（进入后手动触发定位）
```

### 4.2 地图与热力图浏览（发现阶段）

**核心描述**: 展示附近纸条密度，引导用户探索

**实现要点**:
- 默认视图：以用户当前位置为中心，**2km 范围**的地图
- **热力图叠加层**：
  - 颜色梯度：浅蓝（稀疏）→ 蓝（中等）→ 紫（密集）→ 红（热点）
  - 数据源：实时纸条密度聚合（后端按网格聚合，减少传输）
  - 可切换：热力图模式 / 单点 Marker 模式
- **探索引导**：
  - 地图边缘显示"前方 300m 有热点"方向箭头
  - 热点区域点击后放大，显示该区域的纸条数量
- **地点 AGENT 标记**：有 AGENT 的地点显示特殊图标（土地公公头像）
- **品牌投放标记**：商家优惠券纸条显示💰图标

**交互细节**:
- 手势：双指缩放、旋转、倾斜（3D 建筑效果增强沉浸感）
- 点击热力图区域 → 弹出底部 Sheet：显示区域名称 + 纸条数 + "去看看"按钮
- 长按地图任意位置 → 弹出"在此处放纸条？"快捷入口

### 4.3 附近纸条列表/预览（发现阶段）

**核心描述**: 50米内纸条的摘要/数量提示（不揭示全文）

**实现要点**:
- **距离分层展示**：
  - **0-50m（可读区）**: 显示纸条卡片，点击可查看全文
  - **50-200m（预览区）**: 仅显示摘要（前 30 字）+ "走近 123m 查看全文"提示
  - **200m+（探索区）**: 仅显示纸条数量提示（"前方有 5 张纸条"）
- **卡片设计**：
  - 未到达位置：半透明遮罩 + 摘要文字 + 距离提示 + 模糊化内容
  - 已到达位置：完整内容展示 + 阅读计数 + 生命周期进度
- **数量提示**：列表顶部显示 "附近有 12 张纸条，已解锁 3 张"
- **语音纸条标识**：卡片左上角显示 🎙️ 图标 + 时长

**关键组件 `NoteCard`**:
```typescript
interface NoteCardProps {
  note: Note;
  userDistance: number;        // 用户与纸条的实时距离（米）
  isUnlocked: boolean;         // 是否在 50m 内
  isRead: boolean;             // 是否已读过
  onPress: () => void;
}

// 视觉状态：
// - locked + 未读:  模糊背景 + 摘要 30 字 + "还需走近 XX 米"
// - unlocked + 未读: 清晰背景 + 全文 + "点击查看"脉冲动画
// - unlocked + 已读: 灰色背景 + 全文 + "已读"标签
```

### 4.4 纸条搜索/筛选（发现阶段）

**核心描述**: 按类型、情绪、话题筛选纸条

**实现要点**:
- **筛选维度**（底部抽屉或顶部 Filter Bar）：
  - **类型**: 全部 / 文字 / 语音
  - **情绪**: 😊 开心 / 😢 难过 / 😠 愤怒 / 😰 焦虑 / 💕 告白 / 🤔 感悟 / 😂 吐槽
  - **话题**: 自动从内容提取的话题标签（后端 SW-03 NLP 分析）
  - **时间**: 最近 1h / 24h / 7d / 30d
  - **生命周期**: 即将过期（剩余 < 1h）/ 热门（阅读数 > 10）/ 永久
- **搜索框**: 关键词搜索（仅搜索 50m 内已解锁纸条的标题/内容）
- **排序方式**: 最近发布 / 距离最近 / 阅读最多 / 回复最多

**情绪标签 UI**:
- 发布时让用户选择情绪标签（可选，默认"感悟"）
- 筛选栏显示情绪表情图标，选中时有放大动画
- 地图模式可按情绪筛选后重新渲染热力图（"只看告白"模式）

### 4.5 位置验证（核心阶段）

**核心描述**: 确认用户确实在物理位置半径50米内

**实现要点**:
- **实时距离计算**：持续监听 GPS，计算用户与纸条的实时距离
- **50m 解锁机制**：
  - 用户进入 50m 半径 → 卡片从"锁定"变为"解锁"（动画：模糊褪去 + 锁图标打开）
  - 用户离开 50m 半径 → 已读内容仍可查看（本地缓存），未读内容重新锁定
- **距离提示**：
  - 列表页："还需走近 23 米"
  - 地图页：以纸条为圆心的 50m 虚线圆环，用户位置在圆环外时显示指引箭头
- **精度容差**：
  - GPS 精度 < 20m 时严格执行 50m 规则
  - GPS 精度 20-50m 时放宽至 70m（避免室内定位不准导致无法阅读）
- **防作弊**：
  - 检测 Mock 位置（开发者选项 / 模拟定位 App）
  - 速度异常检测（GPS 瞬间跳变 > 500m/s 视为作弊）

**组件 `DistanceBadge`**:
```typescript
// 状态机：
// "far"      (200m+):  "前方 230 米有纸条"
// "approach" (50-200m): "还需走近 67 米"
// "near"     (10-50m):  "已经很近了！再走 12 米"
// "unlocked" (0-10m):   "🎉 已解锁"
// "reading":            "阅读中..."
```

### 4.6 放下纸条（内容创作）（核心阶段）

**核心描述**: 撰写文字或录制语音纸条

**实现要点**:
- **发布入口**：
  - Tab 栏中央 "+" 按钮（最显眼位置）
  - 地图长按快捷发布
  - 纸条详情页 "回复"按钮（属于回复，非新纸条）
- **文字纸条**：
  - 输入框：最大 500 字，实时字数统计
  - 占位符：根据地点类型动态变化
    - 通用："写下你在这里的想法..."
    - 餐厅："这家店怎么样？"
    - 景点："此刻的感受是..."
    - 医院："给同在这里的人一句鼓励..."
- **语音纸条**：
  - 录制界面：按住录音按钮，显示波形动画 + 已录时长（最大 60s）
  - 可预览播放、重录、放弃
  - 语音自动转文字摘要（后端 SW-03），用于列表展示
- **情绪标签选择**：
  - 横向滚动表情选择器：😊 😢 😠 😰 💕 🤔 😂
  - 可选话题标签（自动推荐基于地点的热门话题）
- **位置确认**：
  - 发布前展示地图快照，标注当前位置
  - 提示："纸条将放在这里，50米内的人才能看到"
- **生命周期选择**：
  - 默认 24h，可切换 7d / 永久
  - 说明："越多人读，活得越久"

**发布流程**:
```
点击 "+"
  ├── 选择类型：文字 / 语音
  ├── 输入内容
  ├── 选情绪标签 + 话题标签（可选）
  ├── 确认位置（地图快照）
  ├── 选择生命周期
  ├── 预览纸条卡片效果
  └── 发布 → 成功动画（纸条飞入地图的特效）
```

### 4.7 捡起纸条（内容消费）（核心阶段）

**核心描述**: 阅读/收听附近的纸条

**实现要点**:
- **解锁动画**：首次进入 50m 范围，播放解锁动效（锁打开 + 光效 + 轻微震动反馈）
- **阅读界面**：
  - 纸条卡片全屏展开：内容 + 作者匿名代号 + 发布时间 + 阅读计数
  - 语音纸条：大型播放按钮 + 波形可视化 + 拖拽进度条
  - 底部：回复按钮 + 分享按钮（生成图片分享）+ 举报按钮
- **阅读计数**：用户阅读一次，计数 +1，触发后端延寿逻辑
- **连续阅读**：下滑进入下一张附近纸条（类似抖音竖滑体验）
- **历史记录**：用户可在"我的"页面查看自己读过的所有纸条

**语音播放组件 `VoicePlayer`**:
```typescript
// 状态：
```

### 4.8 匿名回复/对话链（核心阶段）

**核心描述**: 在纸条下形成匿名讨论

**实现要点**:
- **回复入口**：纸条详情页底部固定输入框
- **回复形式**：文字（最大 300 字）/ 语音（最大 30s）
- **对话链展示**：
  - 树状结构：主纸条 → 一级回复 → 二级回复（最多嵌套 2 层）
  - 每条回复显示：匿名代号 + 时间 + 内容
  - 不显示头像，用颜色区分不同用户（同一用户在对话链中保持同一颜色）
- **实时更新**：
  - 用户在看某纸条时，新回复通过 Realtime 实时追加
  - 底部输入框上方显示"X 人正在看此纸条"（Presence）
- **回复通知**：
  - 纸条作者收到推送："有人回复了你的纸条"
  - 回复者收到后续回复通知（可选开启）

**对话链组件 `ReplyThread`**:
```typescript
// 视觉层级：
// 主纸条：大号字体 + 白色背景卡片
// ├── 回复 A：缩进 16px + 浅灰背景 + 左彩色竖线
// │    └── 回复 A1：缩进 32px + 更浅灰背景
// ├── 回复 B：新颜色竖线（表示不同用户）
// └── 回复 C
//
// 交互：
// - 点击回复 → 弹出"回复此人" / "举报"
// - 长按回复 → 复制文字
```

## 5. 接口契约（与相邻模块）

### 上游依赖：SW-02 后端服务
```typescript
// 获取附近纸条（带距离分层）
GET /api/notes/nearby?lat={number}&lng={number}&radius=200&limit=50
Response: { 
  unlocked: Array<Note>,    // 50m 内，可查看全文
  preview: Array<Note>,     // 50-200m，仅摘要
  distant: Array<{lat, lng, count}>, // 200m+ 仅聚合数量
  total: number 
}

// 筛选纸条
GET /api/notes/nearby?lat={number}&lng={number}&filter_type=voice&filter_mood=sad&filter_topic=失恋&sort_by=reads

// 发布纸条
POST /api/notes
Body: { 
  content: string, 
  voiceUrl?: string, 
  lat: number, 
  lng: number, 
  lifespan: '24h'|'7d'|'permanent',
  moodTag: string,        // 情绪标签
  topicTags: string[]     // 话题标签
}

// 阅读纸条（增加阅读计数）
POST /api/notes/:id/read
Body: { readerLat: number, readerLng: number }
Response: { note: Note, replies: Reply[], isFirstRead: boolean }
```

### 下游消费：SW-03 AI 服务
```typescript
// 与地点 AGENT 对话
POST /api/agents/:locationId/chat
Body: { message: string, userId: string, sessionId?: string }
Response: { reply: string, sessionId: string, sources: Array<{noteId: string, snippet: string}> }
```

### 下游消费：SW-04 实时通信
```typescript
// WebSocket 订阅实时回复
channel = supabase.channel('note:' + noteId)
channel.on('broadcast', { event: 'new_reply' }, callback)
channel.on('presence', { event: 'sync' }, callback) // 正在看的人数
```

## 8. 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| GPS 室内漂移导致 50m 半径误判 | 高 | 引入 WiFi/BLE 辅助定位；精度差时放宽至 70m；用户可申诉 |
| 语音录制在低端 Android 上崩溃 | 中 | 降级方案：仅文字；使用 expo-av 兼容性模式 |
| 地图 Marker 过多导致内存溢出 | 中 | 聚合算法（点聚合），只在当前视口加载 |
| 用户 Mock 定位作弊 | 中 | 检测开发者选项、模拟位置 App、GPS 跳变速度 |
